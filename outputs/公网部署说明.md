# Offer Arrester 公网部署说明

当前项目为前端 + API 的轻量 Web 应用，部署目录为 `app/`。

推荐优先配置 `DEEPSEEK_API_KEY`，当前项目已经完成 DeepSeek 实测；如果你想用免费层，也可以改配 `GEMINI_API_KEY`；如果你更想用中文生态模型，也可以改配 `ARK_API_KEY` + `ARK_MODEL` 来接豆包 / 火山方舟。

## Vercel

1. 新建项目并导入当前文件夹。
2. 将 Root Directory 设为 `app`。
3. Framework Preset 选择 `Other`。
4. 在 Environment Variables 中添加 `DEEPSEEK_API_KEY`；或者添加 `GEMINI_API_KEY`；或者添加 `ARK_API_KEY` 与 `ARK_MODEL`。
5. 点击 Deploy。

## Netlify

这版更推荐 Vercel。Netlify 若要稳定支持当前接口逻辑，建议改造成标准 Netlify Functions 结构后再部署。

## 本地联调检查

1. 进入 `app/`
2. 配置 `.env`
3. 执行 `npm start`
4. 打开 `http://localhost:4173`
5. 访问 `http://localhost:4173/api/health` 检查服务状态
